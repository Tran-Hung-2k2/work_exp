# Chapter 6: Partitioning

## 1. Tổng quan

- Partitioning (Sharding) = chia data thành nhiều phần nhỏ
- Mỗi partition lưu trên node khác nhau

### Mục tiêu:

- Scale out (horizontal scaling)
- Tăng throughput
- Giảm load mỗi node

---

## 2. Partitioning & Replication

- Thực tế:
    - Data = partitioned + replicated

### Ví dụ:

- 100GB data → chia thành 10 partitions
- Mỗi partition có 3 replicas

→ Total nodes tăng nhưng:
✔ Performance tốt hơn  
✔ Fault tolerance cao hơn

---

## 3. Partitioning Strategies

### 3.1 Partition by Key Range

#### Ý tưởng:

- Chia theo khoảng giá trị key

Ví dụ:

- A–F → Node 1
- G–M → Node 2

---

#### Ưu điểm:

- Range query rất nhanh
- Data có thứ tự

---

#### Nhược điểm:

- Hotspot (data skew)
    - Ví dụ: user mới → toàn vào cuối range

---

### 3.2 Partition by Hash of Key

#### Ý tưởng:

- Hash(key) → phân phối đều

---

#### Ưu điểm:

- Load balance tốt
- Tránh hotspot

---

#### Nhược điểm:

- Không support range query hiệu quả
- Debug khó hơn

---

### 3.3 Hybrid Approach

- Kết hợp:
    - Hash + Range

→ Ví dụ:

- Hash → shard
- Range → sort trong shard

---

## 4. Handling Skew (Hotspots)

### 4.1 Vấn đề

- Một partition quá nhiều traffic

---

### 4.2 Giải pháp:

- Random suffix key
- Split partition
- Add more nodes
- Application-level sharding

---

## 5. Secondary Indexes

### 5.1 Vấn đề

- Index không còn local nữa (distributed)

---

### 5.2 Two approaches

#### a. Local Index

- Mỗi partition giữ index riêng

✔ Fast  
❌ Query toàn hệ thống phải scan nhiều node

---

#### b. Global Index

- Index riêng biệt, phân tán

✔ Query efficient  
❌ Update phức tạp

---

## 6. Rebalancing Partitions

### 6.1 Khi nào cần?

- Add/remove node
- Data không đều

---

### 6.2 Chiến lược

#### Fixed number of partitions

- Chia trước nhiều partition

✔ Dễ scale  
❌ Waste resource

---

#### Dynamic partitioning

- Split khi quá lớn

✔ Linh hoạt  
❌ Phức tạp

---

#### Consistent Hashing

- Hash ring

✔ Minimal data movement  
✔ Scale dễ

---

## 7. Request Routing

### 7.1 Client → Node nào?

#### Cách 1: Client biết partition

- Smart client

---

#### Cách 2: Router

- Load balancer

---

#### Cách 3: Any node

- Node forward request

---

### Trade-off:

| Approach       | Ưu điểm  | Nhược điểm |
| -------------- | -------- | ---------- |
| Client routing | Fast     | Complex    |
| Router         | Simple   | Extra hop  |
| Forwarding     | Flexible | Overhead   |

---

## 8. Parallel Query Execution

- Query chạy trên nhiều node song song

### Flow:

1. Split query
2. Execute trên từng partition
3. Aggregate result

---

### Ưu điểm:

- Nhanh hơn nhiều

---

### Nhược điểm:

- Coordination phức tạp

---

## 9. Key Trade-offs

| Strategy | Load Balance | Range Query | Complexity |
| -------- | ------------ | ----------- | ---------- |
| Range    | ❌           | ✔           | Low        |
| Hash     | ✔            | ❌          | Medium     |
| Hybrid   | ✔            | ✔           | High       |

---

## 10. Key Insights

- Partitioning = chìa khóa để scale system
- Không có strategy hoàn hảo
- Hotspot là vấn đề phổ biến
- Index + partition = rất phức tạp
- Consistent hashing rất quan trọng

---

## 11. Kết luận

> Replication giúp **chịu lỗi**
> Partitioning giúp **scale**

→ Kết hợp cả hai = hệ thống distributed thực sự
