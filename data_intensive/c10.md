# Chapter 10: Batch Processing

## 1. Tổng quan

- Batch Processing = xử lý dữ liệu theo **lô (batch)**
- Thường chạy:
    - Offline
    - Định kỳ (hourly, daily)

---

### Mục tiêu:

- Xử lý lượng lớn data
- Tối ưu throughput (không phải latency)

---

## 2. Batch vs Online (OLTP)

| Aspect     | Batch Processing | OLTP        |
| ---------- | ---------------- | ----------- |
| Latency    | Cao (phút/giờ)   | Thấp (ms)   |
| Throughput | Rất cao          | Trung bình  |
| Use case   | Analytics, ETL   | Transaction |

---

## 4. MapReduce

### 4.1 Mô hình

#### Map:

- Input → key-value pairs

#### Reduce:

- Aggregate theo key

---

### 4.2 Flow:

1. Read input
2. Map
3. Shuffle (group by key)
4. Reduce

---

### 4.3 Ưu điểm:

- Scale cực tốt
- Fault-tolerant

---

### 4.4 Nhược điểm:

- Verbose
- Không flexible
- Nhiều step → chậm

---

## 14. Key Insights

- Batch = backbone của analytics
- MapReduce đặt nền tảng, nhưng đã lỗi thời
- Spark/Flink là standard hiện đại
- Data locality cực kỳ quan trọng
- Deterministic + retry = fault tolerance
