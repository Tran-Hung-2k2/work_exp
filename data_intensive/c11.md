# Chapter 11: Stream Processing

## 1. Tổng quan

- Stream Processing = xử lý dữ liệu **liên tục (real-time)**

### Khác với batch:

- Data không hữu hạn
- Không có điểm kết thúc
- Xử lý ngay khi data đến

---

### Mục tiêu:

- Low latency
- Real-time insights
- Continuous computation

---

## 2. Stream vs Batch

| Aspect     | Stream     | Batch    |
| ---------- | ---------- | -------- |
| Data       | Unbounded  | Bounded  |
| Latency    | Low        | High     |
| Processing | Continuous | Periodic |

---

## 14. Key Trade-offs

| Aspect      | Stream   |
| ----------- | -------- |
| Latency     | Very low |
| Complexity  | High     |
| Correctness | Hard     |

---

## 15. Key Insights

- Stream = batch vô hạn
- Event time quan trọng hơn processing time
- Exactly-once rất khó đạt
- Log-based systems (Kafka) là nền tảng
- Stateful processing là cốt lõi
